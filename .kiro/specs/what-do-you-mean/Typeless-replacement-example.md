# Example

标题：零成本替代 Typeless：我是怎么用开源 + 免费 API 搞出同样体验的
第一部分：初衷与本质分析

Typeless 一个月 $30，一年 $144（注：图片此处显示一年 $144，可能指优惠价）。功能确实好用——你说话它自动帮你去掉“嗯”“那个”，还能根据你当前在用什么软件自动调整输出格式。在 VS Code 里说话出代码格式，在微信里说话出聊天语气。

但它的本质是什么？就是 Whisper 转录 + GPT 润色。你每个月花的 $30 里，很大一部分是在帮它付 OpenAI 的 API 账单。我决定自己做一个。
第二部分：核心思路——选对模型比什么都重要

做这个东西需要两层模型：

    ASR 层： 负责把你的语音变成文字。

    LLM 层： 负责把文字润色成人话。
    这两层分别选对了，体验就能逼近甚至超过 Typeless。

1. ASR 层：Groq Whisper

    Cerebras 公布的数据显示 Llama 3.1 8B 达到约 1,800 tokens/秒，比 Groq 快 2.4 倍——但这是 LLM 推理速度。

    ASR 这块，Groq 托管的 Whisper Large v3 Turbo 依然是性价比之王。

    每小时转录成本只要 $0.04，速度是实时的 216 倍。一句话说完，0.5 秒文字就出来了。

    而且 Groq 的 API 完全兼容 OpenAI 协议，改一行 base URL 就能接进任何支持 OpenAI 的工具。

    中文场景： 我用的是阿里云 DashScope 的 千问 ASR，中文和方言的准确率确实是最好的。

2. LLM 润色层（重点）：GPT-OSS-120B on Cerebras

这是整个方案里最关键的一环，也是和 Typeless 拉近差距的地方。

    很多人做语音转文字工具，润色那一步要么用 GPT-4o（贵），要么用本地小模型（慢/笨）。我一开始用通义千问 Qwen-Plus，中文效果不错但速度太慢，开了智能模式最慢要一分多钟。而 Typeless 基本上 5 秒以内就出结果。

    后来我发现了 Cerebras。

    GPT-OSS-120B 是 OpenAI 的第一个开源权重推理模型，在 Cerebras 推理云上运行速度高达每秒 3,000 tokens。这个 120B 参数的模型在智能水平上与 Gemini 2.5 Flash 和 Claude Opus 4 等顶级闭源模型持平——但它是 Apache 2.0 开源的。

    而且跑在 Cerebras 的晶圆级芯片上，速度完全不是一个量级。在 GPU 上需要一分钟才能完成的推理任务，在 Cerebras 上只需要一秒。

关于成本：

    Cerebras 跑 GPT-OSS-120B 的输出速度为 2,224 tokens/秒，碾压第二名 SambaNova (647.6 t/s) 和第三名 Together.ai (629.9 t/s)。

    定价为 $0.25/百万输入 tokens 和 $0.69/百万输出 tokens。

    对于语音听写润色来说，每次调用就几十个 token 进出。我算了一下，正常使用一个月成本不到 $0.10。

    而且——Cerebras 开放了免费层，每天 100 万 tokens。你一天能说多少话？100 万 tokens 根本用不完。所以 LLM 润色层实际上是零成本的。

    迁移代码很简单，改掉 OPENAI_BASE_URL，填上免费申请的 Key 即可。

第三部分：除了模型，还做了什么（工程问题）

光有模型不够，Typeless 的体验好在“无感”——按住热键说话，松开就自动出结果。这背后有几个工程问题要解决：
1. 上下文感知

我基于 OpenWhispr 做了一套上下文感知系统。应用会自动检测你当前在用什么软件，然后动态调整 LLM 的 System Prompt。

    VS Code 里： 输出是代码格式，技术术语用反引号包裹。

    微信/飞书里： 输出是轻松的聊天语气。

    Word/Notion 里： 输出是正式书面语。

    实现方式： 用 active-win 这个 npm 包获取当前窗口的 App 名称和标题，然后匹配预设规则表，拼接 System Prompt 传给 LLM。

2. 词典自动学习（解决专有名词痛点）

语音转写最大的痛点之一是专有名词总是错。我做了一个多阶段的词典后处理系统：

    首字母缩写折叠（比如把 "A P I" 合并成 "API"）。

    Token 合并、命名实体纠错、大小写强制。

    自动学习： 系统会对比 ASR 原始输出和 LLM 润色后的文本，自动提取差异词作为候选新词入词典。用得越久，识别越准。

3. 粘贴失败兜底

Typeless 的自动粘贴在 Windows 上有个硬问题：你说完话的时候焦点可能已经不在原来的输入框了。我做了一个兜底机制——如果检测到当前窗口不可输入，就自动弹出一个底部卡片，你手动点一下复制就行，不会丢内容。
第四部分：总结与成本对比
最终月成本对比
项目	Typeless	我的方案
ASR (语音转文字)	包含在 $30 里	Groq Whisper: $0.60/月
LLM (文字润色)	包含在 $30 里	Cerebras GPT-OSS-120B: $0 (免费层)
上下文感知	✅	✅ (自己写的)
词典学习	✅	✅ (自己写的)
月总成本	$30	< $1

结论：一年省下 $348。
怎么自己搞？

项目基于 OpenWhispr 开源项目（GitHub 1400+ stars）做的二次开发，Electron 架构，支持 Windows 和 Mac。核心改动：

    新增上下文感知模块（窗口检测 + 动态 Prompt）。

    新增 DashScope 双链路支持（中文 ASR + Qwen LLM）。

    词典后处理从单点替换升级为多阶段纠错。

    自动学习系统（原文 vs 润色文本差分学习）。

    Windows 文本监控能力扩展（UIA Pattern 覆盖更多控件）。

写在最后：
世界上最强的大模型就那么几个，市面上大部分 AI 产品本质上就是拿这些模型包装一下卖你订阅费。Typeless 的功能确实好，但 $30/月的定价，说白了就是信息差的钱。当开源生态已经足够成熟的时候，你至少应该知道自己有别的选择。
